# Project Overview: PingPong (P Language)

This project is a classic "Ping-Pong" demonstration implemented in the **P Programming Language**. P is a domain-specific language designed for modeling and formally verifying asynchronous distributed systems using state machines.

The project consists of two primary state machines:
- **Pinger:** Sends a `Ping` event to a `Ponger` and waits for a `Pong` event before repeating.
- **Ponger:** Listens for `Ping` events and responds with a `Pong` event.

The system is orchestrated by a `Main` machine that initializes both the `Pinger` and `Ponger`. A `LivenessMonitor` ensures that every `Ping` is eventually followed by a `Pong`.

## Project Structure

- `PingPong.pproj`: Central XML configuration file for the P compiler.
- `PSrc/`: Core state machine definitions (`PingPong.p`).
- `PSpecs/`: System-level specifications (`SystemTests.p`) and liveness monitors (`LivenessMonitor.p`).
- `scripts/`: TypeScript utilities for generating Mermaid visualizations.
- `package.json`: Node.js configuration for dependencies and visualization scripts.
- `PGenerated/`: Artifacts generated by the P compiler.
- `PCheckerOutput/`: Logs and Mermaid diagrams generated by verification and visualization tasks.

## Building and Running

The project uses the P compiler and checker (v3.0.4+).

### Recommended Workflow (Makefile)

- `make setup`: Install Node.js dependencies for visualization.
- `make compile`: Compiles the P project into a .NET DLL.
- `make check`: Runs the P model checker to verify safety and liveness.
- `make trace`: Generates a verbose execution log for visualization.
- `make mermaid`: Converts the execution log into a Mermaid sequence diagram.
- `make graph`: Converts the state machine structure into a Mermaid state diagram.
- `make compile-viz`: Generates XState definitions for interactive animation.
- `make clean`: Removes all generated artifacts.

## Development Conventions

- **State Machine Modeling:** All logic is encapsulated within `machine` or `spec` definitions.
- **TypeScript Integration:** All auxiliary scripts are written in TypeScript and executed via `ts-node`.
- **Formal Verification:** Always run `make check` after logic changes.
- **Visual Validation:** Use `make graph` and `make mermaid` to visually verify behavior.
