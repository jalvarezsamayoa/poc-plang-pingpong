# PingPong: Formal Modeling and Verification in P

This project is a classic "Ping-Pong" distributed system demonstration implemented in the **P Programming Language**. It serves as a blueprint for modeling, verifying, and visualizing asynchronous, event-driven systems using formal methods.

## 🚀 Overview

The system models a reliable handshake between two state machines:
- **Pinger**: Initiates communication by sending `Ping` events.
- **Ponger**: Responds to `Ping` events with `Pong` acknowledgments.
- **LivenessMonitor**: A formal specification (spec) that ensures every `Ping` is eventually followed by a `Pong`, preventing infinite "silent" loops.

The project is integrated with a comprehensive toolchain for **Formal Verification** and **Automated Visualization**.

## ⚙️ Getting Started

### 1. Initialization
Set up the Node.js environment for visualization tools:
```bash
make setup
```

### 2. Compilation
Compile the P source files into a verifiable .NET DLL:
```bash
make compile
```

### 3. Formal Verification
Run the P model checker to prove safety and liveness properties:
```bash
make check
```

---

## 📊 Visualization Workflow

This project supports three distinct levels of visualization:

### 1. Interactive Animation (XState)
Generate an XState-compatible definition to animate your state machine:
1. Run `make compile-viz`.
2. Open `PGenerated/Stately/PingPong.ts`.
3. Click **"Visualize"** in VS Code to interactively trigger events.

### 2. State Machine Diagrams (Mermaid)
Export the static structure of your machines to a Mermaid state diagram:
```bash
make graph
# Output: PCheckerOutput/graph.mermaid
```

### 3. Sequence Interaction Traces (Mermaid)
Capture a "happy path" execution and visualize it as a sequence diagram:
```bash
make trace    # Captures a 10-step execution log
make mermaid  # Converts the log to a sequence diagram
# Output: PCheckerOutput/trace.mermaid
```

## 🧠 Development Conventions

- **State Machine Rigor**: All logic is encapsulated in `machine` or `spec` blocks.
- **Single Source of Truth**: The `.p` files are the master logic; all code and diagrams are generated from them.
- **Liveness First**: Always verify with `LivenessMonitor` to ensure your system doesn't hang in "hot" states.
